## Decisions Made

- **Migration 0008 follows the 0006 / 0001 multi-statement-with-breakpoints pattern**, not one-file-per-statement. Reason: drizzle-kit 0.31 emits ALTER TYPE ADD VALUE statements with `--> statement-breakpoint` separators that the migrator splits and runs each in its own transaction. The 0001 / 0006 precedent confirms this works under advisory lock without "ALTER TYPE … ADD cannot run inside a transaction block" errors.
- **`tests/unit/env.test.ts` was created NEW** (the plan instructed "extend" but the file did not exist). VALIDATION.md row D-32 explicitly allows "NEW or extend", so this is a benign Rule 3 deviation. The 6 Phase 02.2-01 tests are LIVE rather than `it.skip` because the env schema additions land in this same plan — leaving them skipped would be a vacuous-pass per Plan 01-07's vacuous-pass guard pattern.
- **`tests/integration/anonymous-401.test.ts` reserved entries are commented out** rather than added live. The vacuous-pass `toContain` guard would fail if `/api/me/export` were in MUST_BE_PROTECTED before Plan 02.2-03 mounts the route. The comment carries the literal "uncomment after Plan 02.2-03 lands the routes" so the executor for that plan finds it via grep.
