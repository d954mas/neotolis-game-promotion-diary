## Decisions Made

- **Test strategy: SSR-render via svelte/server, not full app.request.** Reasons: (1) the +page.server.ts load functions are env-only — no DB access, no session — so they can be invoked directly with `undefined`; (2) anonymous-200 is structurally guaranteed by PROTECTED_PATHS absence in +layout.server.ts, which we assert as a source-string check (the load-bearing 401-trigger lives there); (3) full app.request through the SvelteKit handler requires `pnpm build` output and is the smoke gate's job. The pattern matches audit-render.test.ts which has been load-bearing in Phase 02.1.
- **`runLoad<Shape>` typing helper.** SvelteKit's PageServerLoad return type is `void | (PageData merge object)`. The shape is well-known for these env-only loaders, so a `runLoad<Shape>(fn)` helper narrows the return to a known interface (PrivacyData / TermsData / AboutData). Cleaner than `// @ts-ignore` per call site.
- **DOMAIN gating on /about.** The canonical-instance link only renders when env.DOMAIN is non-empty. In self-host / CI / dev (where DOMAIN defaults to ''), the link is omitted — which is correct UX, because a self-hosted instance has no separate "canonical SaaS instance" to link to. Preserves D-31 self-host parity without a code branch on APP_MODE (AGENTS-INV-8).
- **Last-updated date as hardcoded ISO string.** Each +page.server.ts has `lastUpdated: "2026-05-01"` baked in. This is a legal-compliance audit point — bumping the date is a deliberate act tied to material policy changes, not an env override. Documented in the +page.server.ts header comments.
- **Magic phrase locking.** "Right to Erasure" (Article 17) and "early access" are asserted as substring matches in the integration test. Future copy edits that break these phrases trip the test loudly — preventing accidental regression of legal-compliance language.

This is a worktree-setup artifact, not a code or plan issue.
