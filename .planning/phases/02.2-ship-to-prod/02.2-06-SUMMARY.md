## Decisions Made

- **`scripts/backup.sh` is the canonical opt-in backup script** for both author-instance (D-25 / R2 + Object Lock) and self-host (D-25a — operator picks the remote in `docs/self-host/backups.md`). Single source-of-truth via env-overridable `RCLONE_REMOTE` / `RCLONE_BUCKET` / `PG_CONTAINER` — no fork between the two modes. Self-host parity preserved (D-31) — the application has zero awareness of backup credentials; rclone reads its own config from `~/.config/rclone/rclone.conf`.
- **OCI image labels live in the runtime stage** of the Dockerfile, not the build stage. Multi-stage builds discard intermediate stages — labels added before the final `FROM node:22-alpine AS runtime` would not survive into the GHCR image. Verified via `docker inspect` mental model.
- **No modification to `docker-compose.selfhost.yml`** — D-31 is the load-bearing CI trust signal (smoke gate boots selfhost.yml). `docker-compose.prod.yml` is a parallel template, not a fork. Verified via `git diff --quiet docker-compose.selfhost.yml` returning 0 before commit.
- **js-yaml chosen over alternative YAML parsers** — pure-JS, no native deps, MIT-licensed, ~4MB unpacked, version-pinned at 4.1.0. The compose-prod tests need actual YAML semantics (not regex), and js-yaml is already a transitive dep of many tools so it adds essentially zero new install weight.
