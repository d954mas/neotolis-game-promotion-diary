## Accomplishments

- **/settings/account page ships the Export + Delete UI verbatim against PLAN §A/B** — Export button calls `fetch('/api/me/export', {method:'GET'})`, builds a Blob URL, programmatically clicks a hidden `<a download="diary-export-YYYY-MM-DD.json">`, and revokes the URL. Delete button opens ConfirmDialog with `requireText="DELETE"`; on confirm, `fetch('/api/me/account', {method:'DELETE'})` then `goto('/login', {invalidateAll: true})`. Both error paths surface via the existing InlineError component.
- **AccountDeletedBanner ships at the root layout (mounted when `data.user.deletedAt` is non-null)** with title + days-left countdown + Restore CTA (POST /api/me/account/restore + invalidateAll). The "Permanent delete now" CTA is HIDDEN — purge endpoint ships in Phase 3 purge worker (PUTOFF comment carries the closing plan number).
- **ConfirmDialog `requireText?: string | null` prop ships as a fully backward-compatible extension** — when null/undefined (every existing call site), no behaviour change. When set (e.g. "DELETE"), renders a text input under the message and disables the confirm button until typedText matches. Layered with `isIrreversible` (the checkbox); both gates must pass when both set. The Phase 2.1 industry-standard speed-bump pattern extended with the GitHub-style Type-DELETE pattern (D-S3).
- **Auth-gated noindex meta via inverse-allowlist** — `<meta name="robots" content="noindex,nofollow">` emitted in `<svelte:head>` for every route NOT in `PUBLIC_INDEXABLE_PATHS` (/, /login, /privacy, /terms, /about). The 5 public surfaces stay indexable; everything else (/feed, /sources, /games, /events, /audit, /keys, /settings, /settings/account) gets the noindex. D-13 invariant satisfied with one prop in one file — no SvelteKit route-group restructure needed.
- **/login disclaimer (D-12) ships as a Paraglide-keyed paragraph** below the Continue with Google CTA: "Early access. Auto-import from registered sources arrives in a future release; manual paste works now." Prevents the "I added a YouTube channel and nothing happened" confusion before Phase 3 polling lands.
- **toUserDto extended with `deletedAt: Date | string | null`** — the projection forwards the new column from the user table to the layout consumer. AGENTS §5 secret-strip invariant unaffected (deletedAt is a timestamp, not a ciphertext column). The Plan 02.2-03 export envelope test continues to assert googleSub / refreshToken / accessToken / idToken / kek_version absent. One new dto.test.ts unit test asserts the deletedAt forwarding works for both null and non-null states.
- **21 Paraglide message keys added** following the `<page>_<concern>_<text>` convention — settings_account_* (12), account_deleted_banner_* (5), confirm_dialog_type_delete_* (2), login_early_access_disclaimer (1), settings_account_link (1). EXPECTED_KEYS keyset snapshot in tests/unit/paraglide.test.ts extended to 264 keys (was 243); locale-add invariant (D-41) preserved — adding a new locale = drop a JSON file matching the keyset.
- **8 live Playwright browser tests** in tests/playwright/account-flows.spec.ts, all PASSING locally:
  - /login disclaimer copy assertion (D-12)
  - 5× public page noindex absence (/, /login, /privacy, /terms, /about — D-13)
  - /settings/account anonymous redirect to /login?next=...
  - post-redirect /login stays indexable (defense-in-depth)
