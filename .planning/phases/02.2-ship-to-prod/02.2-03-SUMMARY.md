## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] api_keys_steam has no deletedAt column — schema does not support soft-cascade text in plan**
- **Found during:** Task 1 (reading src/lib/server/db/schema/api-keys-steam.ts to verify the deletedAt column)
- **Issue:** Plan acceptance criteria says "softDeleteAccount calls `tx.update(games)…tx.update(apiKeysSteam)` — exactly 5 cascade tables". But api_keys_steam.ts schema declares no deletedAt timestamp column (verified line-by-line; D-14 says keys are hard-deleted via removeSteamKey, not soft). The plan reference verbatim text is in conflict with shipped Phase 2 schema reality.
- **Fix:** softDeleteAccount soft-cascades to the 4 tables that DO have deletedAt (games, game_steam_listings, data_sources, events) and HARD-deletes api_keys_steam rows inside the same transaction. Privacy-positive: envelope-encrypted secrets are gone the moment the user requests deletion. restoreAccount does NOT bring keys back; the user re-enters them. Documented prominently in account.ts file header.
- **Files modified:** src/lib/server/services/account.ts (file header + softDeleteAccount + restoreAccount comments), tests/integration/account.test.ts (cascade test asserts api_keys_steam length === 0 after delete, with rationale comment)
- **Verification:** typecheck green, lint green, test file structure compiles. Phase 6 PRIV-04 polish can add deletedAt to api_keys_steam if this turns out to be wrong; for now it's the privacy-positive default.
- **Committed in:** `4cae004` (Task 1 commit)

**2. [Rule 3 - Blocking] mapEventsToDtos signature mismatch in plan reference**
- **Found during:** Task 1 (writing the import block — plan code had `mapEventsToDtos(eventsRows)` 1-arg form)
- **Issue:** Plan reference shows `events: await mapEventsToDtos(eventsRows)` but the actual exported function in src/lib/server/dto.ts is `mapEventsToDtos(userId: string, rows: EventRow[])` (2-arg). Existing call sites in `src/lib/server/http/routes/events.ts`, `routes/games.ts`, and `src/routes/feed/+page.server.ts` all use the 2-arg form.
- **Fix:** Used the 2-arg form (`await mapEventsToDtos(userId, eventsRows)`) — matches every existing caller. Tenant-scope ESLint rule passes because the userId is forwarded to the eventGames junction lookup.
- **Files modified:** src/lib/server/services/account.ts
- **Verification:** typecheck green; lint green.
- **Committed in:** `4cae004` (Task 1 commit)

**3. [Rule 3 - Blocking] mapErr signature mismatch in plan reference**
- **Found during:** Task 2 (writing routes/account.ts — plan code destructured `{status, body} = mapErr(err)` from a 1-arg form returning a value)
- **Issue:** Plan reference shows `const { status, body } = mapErr(err); return c.json(body, status);`. The actual exported function in `_shared.ts` is `mapErr(c: Context, err: unknown, route: string): Response` — it RETURNS the Response directly via `c.json(...)`, not a destructurable object.
- **Fix:** Used the actual signature `return mapErr(c, err, "GET /api/me/export")` — matches every existing caller (sessions.ts, audit.ts, games.ts).
- **Files modified:** src/lib/server/http/routes/account.ts
- **Verification:** typecheck green; lint green.
- **Committed in:** `1430a22` (Task 2 commit)

**4. [Rule 3 - Blocking] anonymous-401 MUST_BE_PROTECTED is a string[], not Array<{method, path}>**
- **Found during:** Task 2 (uncommenting the 3 reserved entries — Wave 0 comment block had drafted them as `{ method: "GET", path: "/api/me/export" }`)
- **Issue:** The MUST_BE_PROTECTED array is a `string[]` of route patterns (Phase 1 + Phase 2 + Phase 2.1 entries are all bare strings). The vacuous-pass guard does `expect(protectedPaths).toContain(required)` where `protectedPaths` is `string[]`. Adding `{method, path}` objects would break the toContain assertion immediately.
- **Fix:** Replaced the comment block with bare string entries (matching the existing array shape) AND added 3 explicit `it("Plan 02.2-03: anonymous METHOD /api/... returns 401")` blocks immediately above the existing AUTH-01 final assertion — both layers per AGENTS.md §3 (vacuous-pass sweep + explicit per-route assertions).
- **Files modified:** tests/integration/anonymous-401.test.ts
- **Verification:** typecheck green; the 3 new explicit per-route assertions PASS locally without DB (they only need the createApp() router; no Postgres hit).
- **Committed in:** `1430a22` (Task 2 commit)
