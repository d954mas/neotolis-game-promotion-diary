**2. [Rule 1 - Bug] AuditRow chip-label regex updated for Svelte 5 SSR scoped class output**
- **Found during:** Task 5 (audit-render.test.ts retargeting)
- **Issue:** The plan's regex `<span class="chip"[^>]*>([^<]*)<\/span>` requires `class="chip"` to be the EXACT class attribute. Svelte 5 SSR adds a scoped class suffix (`<span class="chip svelte-1n17fzr">`), so the regex would never match. This was a pre-existing latent bug in Plan 02.1-11's audit-render test that the alias issue masked (the test never reached the regex).
- **Fix:** Updated regex to `<span class="chip(?:\s[^"]*)?"[^>]*>` — accepts the chip class with or without sibling classes. Same fix applied to the FiltersSheet `<label class="check ...">` regex.
- **Files modified:** tests/integration/audit-render.test.ts
- **Verification:** All 4 audit-render tests pass.
- **Committed in:** c94ebc5 (Task 5)

**3. [Rule 3 - Blocking] /feed/+page.svelte ChipAxis local type + applyFiltersFromSheet show? widening**
- **Found during:** Task 4 (FilterChips/FiltersSheet axis-union extension)
- **Issue:** FilterChips' ChipAxis union widens from `"kind"|"source"|"show"|"authorIsMe"` to add `"action"` for /audit reuse. /feed/+page.svelte has its own local ChipAxis type AND its own sheetFocusAxis state typed against the narrower 4-axis union. Without widening, the /feed component fails typecheck against the new FilterChips contract. Similarly, FiltersSheet's onApply now has `show?: ShowFilter` (was required) so applyFiltersFromSheet must default it.
- **Fix:** Extended /feed local ChipAxis + sheetFocusAxis to include 'action' (defensive — /feed never receives that axis); dismissAxis adds an unreachable `'action'` no-op branch; applyFiltersFromSheet defaults show? to {kind:'any'} when undefined.
- **Files modified:** src/routes/feed/+page.svelte
- **Verification:** Typecheck shows zero new errors in /feed/+page.svelte; existing 5 pre-existing errors (games.test.ts / game-listings.test.ts retired Phase 2 schemas) unchanged.
- **Committed in:** 647b200 (Task 4)

**4. [Rule 3 - Blocking] HTTP route src/lib/server/http/routes/audit.ts also migrated to multi-action shape**
- **Found during:** Task 3 (listAuditPage signature widening)
- **Issue:** The plan's Task 3 only mentions `src/lib/server/services/audit-read.ts` + `src/routes/audit/+page.server.ts`.
