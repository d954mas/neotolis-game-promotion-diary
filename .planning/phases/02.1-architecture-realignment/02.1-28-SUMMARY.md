## Decisions Made

- **createEvent does NOT issue per-game attached_to_game audits.** Initial-set INSERT writes one `event.created` with `metadata.game_ids=[...]`; only `attachEventToGames` issues per-game add/remove audits (covers diff). Audit chain: `event.created` (full set), then per-game `attached_to_game` / `detached_from_game` (deltas).
- **events cascade REMOVED in softDeleteGame.** M:N relations make "delete events whose gameId=this" ambiguous (events can attach to multiple games). Schema CASCADE on `event_games(game_id)` handles HARD-delete; soft-delete leaves the junction alone (no deletedAt column by Plan 02.1-27 design).
- **markStandalone REJECTS attached events** (replaces Plan 02.1-24 silent-detach). 422 `standalone_conflicts_with_game`; user must detach first. UI hides the affordance (Plan 02.1-32) — defense-in-depth.
- **Back-compat alias `{gameId}` accepted at the route layer for one round of UAT.** Plan 02.1-32 retires the alias on the UI side. Both `createEventSchema` and `attachSchema` use `.transform()` to normalize singular → plural.
- **DTO signature extension `(event, gameIds[])`.** Pure projection; no DB I/O. Caller passes the loaded junction; batch loaders (`mapEventsToDtos`) avoid N+1.
- **Stale Plan 02-04 'GAMES-02 soft cascade delete' subtest marked it.skip with superseded annotation.** Body elided (referenced retired schemas). The `gameSteamListings` cascade remains covered by the GAMES-02 restore subtest.
- **Random-suffix uniq() pattern applied to every Plan 02.1-28 test-seed email** (parallel-executor coordination — Plan 02.1-17 precedent).
- **ESLint `.claude/` ignore added.** Sibling agents' worktrees pollute the lint surface; treating them as scratch space matches the GSD orchestrator pattern.
