## Decisions Made

See `key-decisions` in frontmatter — 8 architectural / scoping decisions, the most load-bearing being:

- **Discriminated 'show' axis over orthogonal flags** — When the UI cannot produce conflicting filter combinations (Inbox AND Game=X), the type system should mirror that constraint. Replacing two booleans/lists with one tagged union eliminates a class of bugs at compile time and produces deterministic SQL branches without `if (filters.attached && filters.game)` cross-check.
- **CursorPager retention** — Plan's interface block explicitly says "leave as-is and document" if any non-/feed surface still consumes CursorPager. /audit does. /feed gets infinite scroll; /audit keeps cursor pagination (recovery / forensic flows are different mental models from chronological browsing).
- **UTC-calendar-date grouping** — Cheaper, deterministic, and consistent with how listFeedPage stores occurred_at. Local-tz regrouping is filed as Phase 6 polish if real users complain about boundary anomalies (23:50Z + 00:10Z splits).
- **Plan 02.1-19 fixes /events/new only** — Sibling create-form pages have the same loader-stale bug but are filed as a follow-up todo to keep the PR diff focused on the round-2 UAT-flagged surface.
