## Accomplishments
- **/games/[id] P0 gap closed.** Phase 2 UAT identified: backend supported rename + add-Steam-listing but the UI never composed the affordances. Plan 09 ships RenameInline (click-to-edit `<h1>`) + AddSteamListingForm (Steam URL paste → appId parse → POST /api/games/:gameId/listings).
- **Unified-events curated layout on /games/[id].** The Phase 2 four-panel stack (header / channels / items / events) collapses to the unified-events curated layout (header / store-listings / events grouped by month with FeedRow rows). The deprecated YouTube channels and tracked items panels are gone.
- **/events list page deleted; /events/new is the canonical free-form entry point.** UI-SPEC §"/events (REMOVED)" — events live in /feed (chronological pool) and /games/[id] (curated). The full-page form ships with all 9 event kinds enabled (paste flow is the FAST path; free-form is the FALLBACK).
- **/events/[id] Phase-4 stub.** Per CONTEXT D-07: gives the user a destination for FeedRow's "Open" button without overpromising. Renders a small read-only summary (kind icon + title + timestamp + Open-original link) above an EmptyState explaining VIZ-01 lands in Phase 4.
- **AppHeader → UserChip; theme toggle relocated.** Phase 2 UAT P1 surfaced the toggle was clutter on every page. Plan 09 removes it from the header and adds UserChip (avatar + email + sign-out menu) in its place. ThemeToggle now renders only on /settings.
- **Active sessions list on /settings.** P1 security UX gap closed: users can now see and revoke individual sessions (timestamp + IP + abbreviated UA). The current session is badged; every other row has a "Sign out this session" button calling DELETE /api/sessions/:id.
- **/keys/steam copy polished.** Plan 02.1-03 already rephrased `m.empty_keys_steam_body()` ("Phase 3 lands the manual-entry form" replaces the Phase 2 vaporware mention). The page already consumes the key; Plan 09 adds an integration-test assertion locking the new copy.

## Decisions Made

- **9-kind picker enables ALL 9 kinds.** The plan suggested either disabling pollable kinds (youtube_video / reddit_post) or enabling them all. UI-SPEC fallback note: enabling all 9 keeps the form simple; the paste flow is the FAST path on /games/[id] + /feed, free-form is the FALLBACK. Documented in /events/new/+page.svelte header comment.
- **/events/[id] stub adds a read-only summary.** UI-SPEC FLAG noted "honest, sets expectations, avoids a 404"; the small summary (KindIcon + title + occurredAt + Open-original link) gives the user context above the EmptyState.
- **Active-sessions filter on `expiresAt > now()`.** Stale rows shouldn't render; the user wants to see what's currently signed in.
- **UserChip click-outside + Esc both close the menu.** Two affordances, one mechanism (`document.addEventListener` cleaned up in `$effect`'s return).
- **AppHeader retains the `theme` prop.** Removing it would force a +layout.svelte signature change for no functional benefit; the prop is unused, marked `_theme` to silence lint. Phase 4 cleanup can drop it.
