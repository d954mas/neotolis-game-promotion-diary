The body-scroll-lock is now declarative via CSS :has() in src/app.css:

```css
/* Plan 02.1-34 (UAT-NOTES.md §4.22.F regression fix): declarative body-scroll-
   lock when ANY <dialog> is open. Replaces the imperative
   document.body.style.overflow approach Plan 02.1-22 introduced — that approach
   regressed because Svelte 5 $effect cleanup timing was unreliable + other
   ConfirmDialog instances overwrote the state. CSS :has() is supported in all
   modern browsers (Chrome 105+, Safari 15.4+, Firefox 121+); the project targets
   modern only per stack docs. */
body:has(dialog[open]) {
  overflow: hidden;
}
```

This rule applies the body lock the moment ANY <dialog open> exists in the DOM — Including ConfirmDialog, FiltersSheet, future modals. Self-restoring: when no dialog is open, the rule no longer matches, body returns to default overflow.
