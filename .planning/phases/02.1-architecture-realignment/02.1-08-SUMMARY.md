type NavKey = "feed" | "sources" | "games" | "events" | "audit" | "settings";
const navActive = $derived.by((): NavKey => {

  if (p.startsWith("/feed")) return "feed";

## Decisions Made

- **PollingBadge text is rendered INLINE in `<SourceRow>`** — not via the shared `<PollingBadge>` component Plan 02.1-07 ships. The FeedRow component owns the `role="status"` announce wrapper for chronological-pool state changes; SourceRow only needs the two-state status text (`m.sources_status_auto_on_pending()` / `m.sources_status_auto_off()`) and consumes the keys directly. The decision avoids cross-plan import coupling during parallel execution while yielding identical Phase 2.1 visual output.
- **kindMatrix returned by the `/sources/new` loader server-side**, not re-derived client-side from a shared constant. Plan 02.1-04 keeps `FUNCTIONAL_KINDS` + `KIND_PHASE` private to `services/data-sources.ts`; the loader hand-rolls the 5-element array with i18n key references. Phase 3+ promoting kinds is a loader-only edit (no constant export, no second source of truth).
- **Auto-import default flips OFF on ownership-flip via `$effect`** — the form watches `(!isOwnedByMe && autoImport === initialAutoImport)` and resets `autoImport` to false. Phase 3's polling worker should not run against a blogger channel until the user explicitly opts in. Tracking only the initial-default value (extracted via `untrack` at init) prevents the effect from re-applying after a manual toggle.
- **`m.common_cancel()` on `/sources/new` (not a destructive-cancel scoped key)** — UI-SPEC explicitly chose the global neutral-close key here because typed input is not at risk: navigating back to /sources is the equivalent action. Distinct from the RenameInline `m.game_rename_cta_discard()` destructive-cancel context that Plan 02.1-09 ships.
- **Layout-level changes are coupled into this plan** — the Nav reshuffle is structurally meaningless without the layout's `NavKey` union widening (TypeScript compile error otherwise) and `PROTECTED_PATHS` update (anonymous `/sources` access would render the page instead of redirecting). Three coupled edits land in one commit; splitting across plans would leave an incoherent intermediate state.
- **`tests/browser/responsive-360.test.ts` updated as a doc-only edit** — the deferred `it.skip` list reads the Phase 2.1 surface set (drops `/accounts/youtube`, adds `/feed` + `/sources`). The list is documentation only (assertions stay deferred until Phase 6 lifts the cookie-injection harness); keeping it accurate is the load-bearing piece for future executors who consult the file when the harness lands.

### Auto-fixed Issues

**1. [Rule 1 — Bug] `editName` $state captured initial source.displayName as constant**
- **Found during:** Task 1 verification (`pnpm exec svelte-check`).
- **Issue:** `let editName = $state(source.displayName ?? "")` reads `source.displayName` once at init. A parent passing a freshly-renamed source after the row had been re-rendered would still show the stale value when entering edit mode. Svelte 5's `state_referenced_locally` warning correctly flagged this.
- **Fix:** initialise `editName` to `""`, re-seed from `source.displayName` at the moment edit mode toggles open (`onclick={() => { if (!editing) editName = source.displayName ?? ""; editing = !editing; }}`). The buffer now reflects the latest props every time the form opens.
- **Files modified:** `src/lib/components/SourceRow.svelte`.
- **Commit:** folded into the Plan 08 commit (the fix landed before any commit).

**2. [Rule 1 — Bug] `data.default*` reads on `/sources/new` triggered the same `state_referenced_locally` warning**
- **Found during:** Task 1 verification.
- **Issue:** `let isOwnedByMe = $state(data.defaultIsOwnedByMe)` reads `data` (a reactive `$props()` object) once at init. The form is one-shot (CONTEXT D-09 full-page form), so the reactive read is intentional — but Svelte's warning is conservative.
- **Fix:** extract via `const initialIsOwnedByMe = untrack(() => data.defaultIsOwnedByMe)` (and analogous for `initialAutoImport`). `untrack` decouples the read from the reactive graph so Svelte recognises the intent.
- **Files modified:** `src/routes/sources/new/+page.svelte` (added `import { untrack } from "svelte"`).
- **Commit:** folded into the Plan 08 commit.
