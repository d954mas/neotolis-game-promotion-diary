## PollingBadge unified contract (CONTEXT D-05)

```svelte
<script lang="ts">
  let { event }: { event: { kind: EventKind; lastPolledAt: Date|string|null } } = $props();
  const visible = $derived(
    (event.kind === "youtube_video" || event.kind === "reddit_post") &&
      event.lastPolledAt === null,
  );
</script>
{#if visible}
  <span class="polling-badge" role="status">{m.polling_badge_phase3_placeholder()}</span>
{/if}
```

The check is INDEPENDENT of `event.sourceId` — manually-pasted YouTube videos (sourceId=NULL) get the same badge as source-attached events (sourceId=X). Phase 3 polling worker honors this in `pollStats(event, source)` with nullable `source` (RESEARCH §2.1 + Plan 02.1-05's createEventFromPaste shape).

## AttachToGamePicker fetch flow

```ts
// Attach (game OR null) → PATCH /api/events/:id/attach
const res = await fetch(`/api/events/${event.id}/attach`, {
  method: "PATCH",
  headers: { "content-type": "application/json" },
  body: JSON.stringify({ gameId }),  // gameId can be null
});
// 404 → m.feed_attach_error_game_not_found
// 422 → m.feed_attach_error_already_attached({title})
// other !ok → m.error_server_generic

// Dismiss-inbox → PATCH /api/events/:id/dismiss-inbox
const res = await fetch(`/api/events/${event.id}/dismiss-inbox`, { method: "PATCH" });
// 422 → m.feed_dismiss_error_not_in_inbox (event already attached)
// other !ok → m.error_server_generic
```

After every successful PATCH, the picker calls `onChanged?.()` which the parent FeedRow's parent (the /feed page) wires to `invalidateAll()` — the loader re-runs and the row re-renders with the new gameId / metadata.
