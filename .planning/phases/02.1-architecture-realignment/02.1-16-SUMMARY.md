## What Shipped

### Task 1 — formatFeedDate utility + 6 unit tests (Gap 5 / P1)

- **`src/lib/util/format-feed-date.ts`** — 4-bucket compact relative date helper. Buckets:
  - same calendar day → `"Today, HH:MM"` (zero-padded local-time hours/minutes)
  - one calendar day back → `"Yesterday"` (no time — coarser by design)
  - same calendar year, older than yesterday → `"MMM D"` via `Intl.DateTimeFormat` (`"Apr 25"`)
  - earlier years → `"MMM D, YYYY"` (`"Apr 25, 2025"`)
  - invalid / null / undefined → `"—"` (defensive — UI never renders "Invalid Date")
- **`tests/unit/format-feed-date.test.ts`** — 6 deterministic cases via `vi.setSystemTime` covering all 4 buckets, ISO string input, and the 4 invalid-input branches.

Pure utility — no DB, no env, no network. The tests run in the unit project (~5s feedback latency, well inside the 02.1-VALIDATION.md sampling rate).

### Task 2 — <FeedCard> replaces <FeedRow> on /feed and /games/[id] (Gap 3 + 6, P0+P1)

- **`src/lib/components/FeedCard.svelte`** (new). Visual contract:

## Why It Matters

- **Gap 3 (P0):** the user explicitly rejected the FeedRow text-row design ("хочется видеть фид как карточки"). The feed is the new primary navigation — getting the visual right matters more than spec-adherence when the user is the design authority. Media cards with thumbnails make a 50-row feed actually scan-able.
- **Gap 5 (P1):** full ISO datetimes were visually loud per UAT. Compact buckets ("Today, 14:30" / "Yesterday" / "Apr 25" / "Apr 25, 2025") get the user from "when did this happen?" to "what mattered yesterday?" without parsing.
- **Gap 6 (P1):** the prior KindIcon was decorative-only with no programmatic identity. Path b (text label adjacent) is unambiguous for a11y tooling, scales to all 10 kinds without per-kind icon refactor, and avoids the brand-mark licensing question entirely.
- **Phase-level sign-off:** with the paraglide invariant restored and the per-task map filled, Plan 02.1-10's UAT checkpoint can resume against a fully verified planning artifact.
