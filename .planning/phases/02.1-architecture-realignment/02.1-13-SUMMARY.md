## Decisions Made

- **Gap 8 chose option (a) over option (b)**: explicit "today" default + inline explainer instead of URL-driven auto-population with a badge. Rationale: YouTube oEmbed has no publish-date field, so option (b) would require the YouTube Data API v3 integration that lands in Phase 3 alongside KEYS-01 — out of scope for 2.1. Option (a) is the honest representation of the actual contract.
- **Multi-control field idiom**: switched the date block from `<label class="field">` (label-as-wrapping-element) to `<div class="field">` + a nested `<label for="event-date">` because `<label>` semantically wraps a single form control. Chips + input + hint are siblings inside the `.field` div; the explicit `for="event-date"` association keeps screen-reader behaviour identical.
- **Snapshot test drift** logged for Plan 02.1-16 (the snapshot-refresh plan), not refreshed here — matches Plans 02.1-01 / 02.1-12 / planned-12-onwards convention.
