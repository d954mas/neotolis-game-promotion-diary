## Decisions Made

See `key-decisions` in frontmatter — 8 decisions, the most load-bearing being:

- **FeedCard is a pure preview tile** — UAT round-2 explicit: "FeedCard with NO action buttons (purely visual + clickable)". AttachToGamePicker is the exception because it's the inbox-clear primary flow (FEED-01 / INBOX-01) and non-destructive.
- **Phase-4 stub replaced, not removed** — CONTEXT D-07 spirit ("honest, sets expectations, avoids a 404") preserved by anchoring the LayerChart placeholder inline at the bottom of the detail page. Future VIZ-01 lands in the same slot.
- **Edit-form gameId mutation stays on /api/events/:id/attach** — keeps the existing separation of concerns. Widening updateEventSchema would fork the cross-tenant gameId validation; the dedicated endpoint already validates ownership via assertGameOwnedByUser.
- **getEventById opts.includeSoftDeleted is opt-IN** — default behavior unchanged so existing callers compile + behave the same. Cross-tenant scope preserved by query construction (userId WHERE clause is independent of the opts flag).
