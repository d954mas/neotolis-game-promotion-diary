## Decisions Made

- **Final keyset is 156, not the plan's estimated ~133.** The plan FLAG said "approximately 53 new keys" but the explicit list in the plan's Step A.2 totals 77 new keys. Phase 2 baseline was 81 (test asserted 81; not the plan's stated 80). The acceptance contract is "the test asserts what the executor ships" — the keyset count is whatever it lands on, not a target number.
- **`events_cta_new_event` kept alongside new `feed_cta_new_event`.** The plan removes only `audit_action_item_*`. Even though `/events` page is being removed in Wave 2, leaving the key avoids a deletion the plan doesn't authorize. Phase 6 housekeeping owns eventual cleanup.
- **Adapter STUB throws via `Error()` (not a typed error class).** The acceptance criterion is "throws Error with message containing X"; using a richer `AppError` would over-spec for a STUB and require importing the AppError module (violates "adapters are pure"). Stay minimal; Phase 3's real implementation can choose its error shape.
- **Single commit for Task 2 (combined test + impl) instead of separate RED → GREEN.** A RED commit that imports a non-existent module is a build break, not a useful RED state — the test fails to even compile, which is louder than the assertion failure. Phase 1 / Phase 2 TDD precedent shows several plans collapse RED+GREEN when the impl-and-test land together.
