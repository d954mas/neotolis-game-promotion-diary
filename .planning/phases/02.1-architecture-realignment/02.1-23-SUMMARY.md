## Decisions Made

- **Inbox label rendered inline in the overlay (option b).** The plan's <interfaces> block named two options for the Inbox indicator inside the new overlay context: (a) extend InboxBadge with a `variant="overlay"` prop or (b) render inline `<span class="overlay-inbox">{m.inbox_badge()}</span>`. Option (b) ships because the overlay needs a dark-pill style consistent with overlay-kind / overlay-mine, and threading a variant prop through InboxBadge for a single consumer is premature abstraction. InboxBadge stays exported for potential future contexts.
- **No new Paraglide keys.** The existing m.inbox_badge() ("Inbox") and m.feed_card_author_is_me_badge() ("Mine") fit the new overlay context unchanged. The plan flagged this as the preferred outcome ("Otherwise NO new keys (preferred)"). EXPECTED_KEYS snapshot stays in lock-step (D-41 invariant preserved).
- **Auto-derived images only for round-3.** Manual upload UI is OUT OF SCOPE per the plan's explicit scope block. A TODO comment near the thumbnailUrl $derived block documents the Phase 3+ schema shape (cover_url column or metadata.image.url jsonb path; resolver returns manual upload OR auto-derived per kind) so the future plan picks up cleanly.
- **Mine treatment = C + A combined (not one-or-the-other).** User explicitly chose this combination during round-3 UAT 2026-04-29. A single visual cue was rejected. Plan 02.1-25 will mirror the same C + A pattern on SourceRow.svelte.
