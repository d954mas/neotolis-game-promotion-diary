## Pitfall 4 mitigation: assertGameOwnedByUser BEFORE update

```ts
async function assertGameOwnedByUser(userId, gameId): Promise<void> {
  const [row] = await db.select({id: games.id}).from(games)
    .where(and(eq(games.userId, userId), eq(games.id, gameId), isNull(games.deletedAt))).limit(1);
  if (!row) throw new NotFoundError();
}

export async function attachToGame(userId, eventId, gameId, ip, ua?) {
  if (gameId !== null) await assertGameOwnedByUser(userId, gameId);   // BEFORE update
  const [row] = await db.update(events).set({gameId, updatedAt: new Date()})
    .where(and(eq(events.userId, userId), eq(events.id, eventId), isNull(events.deletedAt)))
    .returning();
  if (!row) throw new NotFoundError();
  // audit ...
}
```

## INGEST-03 author_is_me inheritance match key

Match: `oEmbed.author_url == dataSources.handleUrl` exact case-sensitive equality. The `findActiveSourceByHandleUrl` helper in events.ts:

```ts
async function findActiveSourceByHandleUrl(userId, handleUrl) {
  const [row] = await db.select().from(dataSources)
    .where(and(eq(dataSources.userId, userId), eq(dataSources.handleUrl, handleUrl), isNull(dataSources.deletedAt)))
    .limit(1);
  return row ?? null;
}
```

Soft-deleted sources do NOT match (the `isNull(dataSources.deletedAt)` clause). Asserted by `tests/integration/ingest.test.ts > INGEST-03: ... soft-deleted source does NOT inherit`.

### Auto-fixed Issues

**1. [Rule 1 — bug] Initial `dismissFromInbox` jsonb_set call did not create the intermediate `inbox` parent**
- **Found during:** Task 1 verification (integration tests).
- **Issue:** `jsonb_set(metadata, '{inbox,dismissed}', 'true', true)` left `inbox` undefined when `metadata` did not already contain it. Postgres's `create_missing=true` flag only creates the LAST key; intermediate parents must already exist.

**2. [Rule 1 — bug] Initial `listFeedPage` tripped the ESLint TENANT_TABLES rule**
- **Found during:** `pnpm eslint src/lib/server/services/events.ts`.
- **Issue:** I built a `whereParts` array including `eq(events.userId, userId)` and spread it into `.where(and(...whereParts, cursorClause))`. The structural rule does textual analysis: it requires `userId` to appear LITERALLY inside `.where(...)`. The spread broke the regex match.
- **Fix:** Separated the userId clause from the filter array — `.where(and(eq(events.userId, userId), ...filterParts, cursorClause))`. The userId clause is now literally inside `.where(...)` and the rule recognizes it. Functionally identical; only the source-text shape changed.
- **Verification:** `pnpm eslint src/lib/server/services/events.ts` exits 0.
- **Committed in:** `6865843`.
