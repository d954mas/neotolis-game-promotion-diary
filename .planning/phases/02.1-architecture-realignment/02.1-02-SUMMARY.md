## Issues Encountered

**Parallel-executor index race (operationally handled, no plan deviation).** A sibling executor agent (presumably the Plan 02.1-01 schema baseline collapse executor) had already pre-staged 3 schema deletions (`game-youtube-channels.ts`, `tracked-youtube-videos.ts`, `youtube-channels.ts`) and several modifications in the working tree's index when this executor ran `git add` for its 5 files. The first commit attempt absorbed those staged deletions. Resolution: `git reset --soft HEAD~1` to undo the commit while preserving index state, then `git commit -o tests/integration/...test.ts tests/browser/feed-360.test.ts -m '...'` to scope the commit to exactly this plan's pathspecs. Sibling agent's index state preserved; commit `1dd40e1` contains only the 5 new placeholder files.

This pattern — `git commit -o <pathspec>...` for parallel-executor scope — is documented above under `patterns-established` so future Wave 0 executors can avoid the same pitfall.
