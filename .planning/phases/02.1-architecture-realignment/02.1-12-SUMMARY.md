# Phase 02.1 Plan 12: Add generic `post` event kind Summary

Closes VERIFICATION.md Gap 12 (P1) — adds a generic `post` event kind for Mastodon / LinkedIn / Bluesky / Threads / unmapped platforms. Resumes strict forward-only migration discipline (CONTEXT D-04 / DV-2): `drizzle/0001_add_post_kind.sql` is the FIRST migration after the Phase 2.1 baseline collapse.

## What Shipped

**Migration:** `drizzle/0001_add_post_kind.sql` — single statement `ALTER TYPE "public"."event_kind" ADD VALUE 'post';--> statement-breakpoint`. No transaction framing (Postgres rejects ALTER TYPE inside a tx). `drizzle/meta/_journal.json` extended with idx=1 / tag=`0001_add_post_kind`. `drizzle/meta/0001_snapshot.json` auto-emitted by `pnpm db:generate`. `pnpm db:check` exits 0.

**Schema / service / route / DTO mirror (Pitfall 6):**

- `src/lib/server/db/schema/events.ts` (+5 LOC) — `eventKindEnum` array gains `"post"` after `"other"`.
- `src/lib/server/integrations/data-source-adapter.ts` (+1 LOC) — `EventKind` union gains `| "post"`.
- `src/lib/server/services/events.ts` (+1 LOC) — `VALID_EVENT_KINDS` const extended.
- `src/lib/server/http/routes/events.ts` (+1 LOC) — zod `eventKindEnum` extended.
- `src/lib/server/dto.ts` (+1 LOC) — `EventDto.kind` union extended.

All 5 mirrors in lock-step. Existing Pitfall-6 test in `tests/integration/events.test.ts` (`VALID_EVENT_KINDS mirrors the pgEnum eventKindEnum.enumValues exactly`) continues to assert equality post-extension.

**UI surface:**

- `src/lib/components/KindIcon.svelte` (+7 LOC) — new `{:else if kind === "post"}` branch BEFORE the `{:else}` "other" fallback. Paper/document silhouette: rounded rectangle with folded-corner polyline + two horizontal lines for "text rows" (geometric, no brand mark).
- `src/lib/components/FilterChips.svelte` (+2 LOC) — `kindLabel` switch handles `case "post"`.
- `src/lib/components/FiltersSheet.svelte` (+1 LOC) — `<option value="post">` appended.
- `src/routes/events/new/+page.svelte` (+2 LOC) — `EventKind` type alias and `KINDS` picker array extended.

**Local consumers extended (Rule 3 auto-fix):**

- `src/lib/components/FeedRow.svelte` — local EventKind union (`+1 LOC`).
- `src/lib/components/PollingBadge.svelte` — local EventKind union (`+1 LOC`).
- `src/routes/events/[id]/+page.svelte` — local EventKind union (`+1 LOC`).
- `src/routes/games/[gameId]/+page.svelte` — local EventKind union (`+1 LOC`).

These 4 components define their own narrow EventKind type (mirroring EventDto). Without the extension, TypeScript would reject API responses where `kind === "post"`. Plan listed only 6 UI consumers explicitly; the 4 additional local unions surfaced during a sweep (per `Pitfall 6` spirit — every TypeScript narrowing site must mirror the 10-kind shape).

**Paraglide:** one new key `event_kind_label_post: "Post"` appended to `messages/en.json` after `event_kind_label_other`. Consumed by `KindIcon.svelte` (no — geometric icon only), `FilterChips.svelte`, `FiltersSheet.svelte`, `/events/new`. Compiled output regenerated by `pnpm paraglide:compile`.
